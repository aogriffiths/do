Rock4 SE build v3 (using rbuild)

  • 23 April 2023
  • rock

WORK IN PROGRESS. THESE INSTRUCTIONS ARE INCOMPLETE AND NOT WORKING. #

The build of the system image is done using

  • radxa-build/rock-4se and
  • radxa-repo/rbuild using docker.

The official build process uses github workflows, e.g. radxa-build/rock-4se uses actions from radxa-repo/rbuild. So it should be possible to clone both repositories, customise them and use GitHub to build a custom image.

However, this page is hopefully going to document how you can build on your local machine.

Vagrant EXPERIMENT #

  1. Install vagrant

    brew install hashicorp/tap/hashicorp-vagrant

  2. Instal VMWare Fusion Player

    It took me a while to do this. Could use virtualbox as an alternative but virtualbox support for Apple Silicon is still beta. Look at the official vagrant documentation, which indicates better support for VMware, also also here

    • Download and install Fusion 13 Player for macOS 12+ from VMWare
    • Download and install Vagrant vmware Utility from Vagrant
    • vagrant plugin install vagrant-vmware-desktop

  3. Create a debos docker image, with some customisations

    This will allow you to get a bash shell with a builduser later on.

    docker build -t rbuild:6 .

3. Open a bash shell using the image and volume
   ```bash
   docker run --rm -it -v "$(pwd)":/root/host --mount source=rock-4se-rbuild,target=/home/builduser/rock-4se-rbuild --tmpfs /dev/shm:rw,nosuid,nodev,exec,size=4g --security-opt label=disable --entrypoint=/bin/bash rbuild:6

  1. Clone the rbuild source Inside the docker container run:

    git clone https://github.com/radxa-repo/rbuild ./

  2. build the OS image

    ./rbuild
